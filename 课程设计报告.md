# 基于BERT的金融新闻标题情感分析

## 1 引言

金融市场的价格波动与新闻事件密切相关。一条政策利好或利空消息，往往能在短时间内左右投资者的情绪，进而影响股价。对金融新闻进行自动化情感分析，可以帮助投资者更快地捕捉市场风向。

传统的情感分析方法大多依赖词典匹配或传统机器学习模型。词典方法在面对一词多义时几乎束手无策，而像SVM、朴素贝叶斯这类模型又难以捕捉句子的上下文语义。BERT自2018年提出以来，在多项NLP任务上刷新了榜单，它通过双向Transformer编码器学习词语在具体语境中的含义，正好解决了传统方法对上下文不敏感的问题。

本实训选择中文金融新闻标题作为研究对象，使用BERT预训练模型进行微调，构建一个二分类情感分析系统，并开发Web应用将分析结果以可视化方式呈现。

## 2 需求分析

### 2.1 具体问题

给定一条中文金融新闻标题，判断其传达的情感倾向是积极还是消极。以"公司预计全年净利润同比增长50%"为例，应将其识别为积极；而"公司收到证监会立案调查通知书"则应判为消极。

输入：一条中文金融新闻标题，例如"海能达与非洲某国公共安全客户签订千万美元项目合同"。

输出：情感类别（积极/消极）、置信度分数、买卖信号强度。例如上述输入应输出"积极，置信度99.2%，强烈看多信号"。

### 2.2 完成的功能

最终系统通过Web页面提供以下功能：

（1）**单条预测**：输入框输入新闻标题，系统返回情感标签、置信度仪表盘、买卖信号。

（2）**批量预测**：多行输入标题，一次性返回全部结果。

（3）**样例数据浏览**：基于本地数据集生成金融新闻样例，附带板块情绪热力图和累计情绪趋势图。

（4）**热点新闻分析**：从新浪财经实时抓取最新金融新闻，用模型预测每条新闻的情感倾向，汇总给出市场情绪信号。

（5）**数据看板**：展示7575条数据的标签分布饼图、文本长度直方图、正负面词云。

（6）**训练历程**：交互式图表呈现训练过程中的Accuracy/F1曲线、混淆矩阵、ROC曲线。

（7）**数据集浏览**：分页展示本地数据集，对每条数据用模型做预测并标注正确与否。

（8）**文件批量分析**：上传CSV或Excel文件，系统逐条预测情感并生成带结果的下载文件。

（9）**模型信息**：展示BERT模型架构和训练参数。

## 3 数据采集与预处理

### 3.1 数据来源与格式

数据集为`train_dict.txt`文本文件，每行是一条JSON格式记录：

```json
{"title": "恒瑞医药注射用卡瑞利珠单抗主要研究终点结果达到方案预设优效标准", "label": "1"}
{"title": "特发信息子公司与特殊机构客户签订亿元军品采购框架协议", "label": "1"}
```

每条记录包含两个字段：`title`（新闻标题，string）和`label`（情感标签，"0"表示消极，"1"表示积极）。

### 3.2 数据统计

将txt转为Excel后统计：

| 指标 | 数值 |
|------|------|
| 总样本数 | 7575 |
| 正面样本（label=1） | 5491（72.5%） |
| 负面样本（label=0） | 2084（27.5%） |
| 平均文本长度 | 约38字 |
| 最长文本 | 约200字 |
| 最短文本 | 约5字 |

标签分布存在一定的不平衡，正面新闻约为负面新闻的2.6倍。这种比例基本符合真实金融市场的新闻分布——上市公司的正面公告确实多于负面公告。

### 3.3 预处理

数据预处理包括以下步骤：

（1）将txt文件中的JSON行逐行解析，使用`random.shuffle()`随机打乱顺序。

（2）按7:1.5:1.5的比例分层抽样，划分为训练集（5301条）、验证集（1137条）和测试集（1137条）。分层抽样保证三个子集中正负样本比例基本一致。

（3）使用`bert-base-chinese`的分词器（tokenizer）将文本转为token IDs。对于超过128字的标题截断处理，不足128字的用[PAD]补齐，同时生成attention mask标记有效位置。

（4）标签直接使用0/1整数值，不做one-hot编码，因为BERT的分类头内部使用CrossEntropyLoss，它期望的是类别索引而非one-hot向量。

### 4.2 模块调用关系

`app.py`是入口，负责路由分发。用户访问主页时返回`templates/index.html`。前端页面通过Ajax请求各API端点，API端点在`app.py`中定义，每个端点调用对应的工具模块：

- `/api/predict` → `model/predict.py` → `SentimentPredictor.predict()`
- `/api/news/realtime` → `utils/news_fetcher.py` → `fetch_real_news()`
- `/api/news/sample` → `utils/news_fetcher.py` → `fetch_sample_news()`
- `/api/data/statistics` → `utils/data_loader.py` → `get_data_statistics()`
- `/api/visualization/*` → `utils/visualization.py` → 各生成函数
- `/api/upload_analyze` → `model/predict.py` → `predict_batch()`

模型训练独立于Web服务，通过`model/train.py`单独运行。训练完成后模型文件保存在`model/saved/`目录，Web服务启动时自动加载。

## 5 详细设计与实现

### 5.1 BERT模型原理

BERT的全称是Bidirectional Encoder Representations from Transformers。它的核心创新是用双向Transformer编码器替代了传统的单向语言模型。

Transformer编码器由多头自注意力机制和前馈神经网络堆叠而成。自注意力的计算公式为：

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

其中Q、K、V分别代表查询、键和值矩阵，$d_k$是键向量的维度。除以$\sqrt{d_k}$的目的是防止点积过大导致softmax梯度消失。

BERT在预训练阶段使用两个任务：Masked Language Model（随机遮盖15%的token让模型预测）和Next Sentence Prediction（判断两句话是否连续）。通过在海量语料上完成这两个任务，BERT学到了丰富的语言知识。

微调阶段，在BERT输出的[CLS]标记向量后接一个全连接层做分类。bert-base-chinese有12层Transformer、768维隐藏层、12个注意力头，总参数量约1.1亿。

### 5.2 模型构建与微调

本项目选择的预训练模型是Google发布的`bert-base-chinese`，专门针对中文语料进行了预训练。

**分类头设计**：取出BERT最后一层的[CLS]池化向量（768维），经过一层dropout（p=0.1）后送入线性分类器`nn.Linear(768, 2)`，输出两个logits分别对应消极和积极的得分，最后通过softmax转为概率。

**超参数设置**：

| 参数 | 值 | 说明 |
|------|-----|------|
| max_seq_len | 128 | 新闻标题较短，128字足够覆盖99%的样本 |
| batch_size | 32 | RTX 4060 8GB显存的最大批次 |
| learning_rate | 2e-5 | BERT微调常用学习率 |
| num_epochs | 10 | 配合早停，实际在第7-8轮停止 |
| warmup_ratio | 0.1 | 前10%步数线性预热 |
| weight_decay | 0.01 | L2正则化 |
| early_stopping | patience=3 | 连续3轮eval_f1不提升即停止 |

**训练设备**：NVIDIA GeForce RTX 4060 Laptop GPU（8GB显存），训练耗时约5分钟。

### 5.3 核心代码解析

**模型定义**（model/predict.py核心逻辑）：

使用HuggingFace的`AutoModelForSequenceClassification`，它自动在BERT主体后添加分类头。加载时指定`num_labels=2`即可获得一个二分类BERT模型。

**分词与数据预处理**：

```python
encoding = tokenizer(
    text,
    max_length=128,
    padding='max_length',
    truncation=True,
    return_tensors='pt'
)
```

`padding='max_length'`会将不足128的序列补零，`truncation=True`则截断超长序列。返回的字典包含`input_ids`和`attention_mask`两个张量。

**预测推理**：

模型输出logits后，用`torch.softmax(logits, dim=1)`得到概率分布，取`argmax`得预测类别。同时计算买入/卖出信号：自信度超过0.8的积极预测输出"强烈看多"，超过0.8的消极预测输出"强烈看空"。

**实时新闻抓取**：

从新浪财经的公开API接口`feed.mix.sina.com.cn`获取最新滚动新闻。该接口返回JSON格式数据，每条包含标题、时间戳和URL。不需API密钥，直接用requests库即可访问。若网络不通，自动降级为本地模拟数据。

### 5.4 结果评估与分析

训练过程采用早停策略，在第8个epoch后验证F1不再提升，训练停止。最终在测试集上的结果：

| 指标 | 数值 |
|------|------|
| Accuracy | 97.63% |
| Weighted Precision | 97.67% |
| Weighted Recall | 97.63% |
| Weighted F1 | 97.64% |
| AUC | 0.9972 |

分类报告如下：

| 类别 | Precision | Recall | F1-Score | Support |
|------|-----------|--------|----------|---------|
| 消极(0) | 0.9722 | 0.9658 | 0.9690 | 313 |
| 积极(1) | 0.9791 | 0.9820 | 0.9805 | 824 |

从混淆矩阵来看，313个真实消极样本中302个被正确分类，11个被误判为积极；824个真实积极样本中809个正确，15个误判。两类错误率都很低，模型在正负样本上的表现均衡。

ROC曲线的AUC值达到0.9972，说明模型区分正负样本的能力极强。预测概率分布图显示，正负样本的预测概率明显分离，大多数样本的置信度在0.85以上，只有少量边界样本置信度较低。

### 5.5 可视化展示

系统通过ECharts交互式图表展示模型各方面的表现：

（1）**训练曲线**：三个子图分别展示训练Loss、验证Accuracy和F1随step的变化。训练Loss平滑下降，验证Accuracy和F1稳步上升，没有明显过拟合。

（2）**混淆矩阵**：热力图直观展示每一类的正确预测数和错误预测数。

（3）**ROC曲线**：TPR-FPR曲线紧贴左上角，说明模型有接近完美的分类能力。

（4）**词云**：正面新闻高频词包括"增长"、"合作"、"签约"、"中标"等；负面新闻高频词包括"减持"、"立案"、"亏损"、"下滑"等。词云结果与金融领域常识一致。

（5）**板块情绪热力**：根据新闻内容自动匹配行业板块，计算各板块的净情绪得分，红柱代表看多、绿柱代表看空。

（6）**预测仪表盘**：半圆仪表盘实时显示置信度，红色到绿色渐变直观反映情感倾向。

## 6 总结

本项目完成了基于BERT的金融新闻标题情感分析系统的全流程开发，包括数据预处理、模型训练、Web应用部署三个环节。

在技术方面，选择了bert-base-chinese作为预训练模型，在7575条金融新闻标题上进行微调。测试集准确率为97.63%，AUC为0.9972，说明BERT对中文金融文本的情感识别能力很强，即使在小规模数据集上也能达到较好的效果。

在工程方面，构建了完整的Flask Web应用，提供了从单条预测到批量分析的多种功能，并通过ECharts实现了训练历程、混淆矩阵、词云等多维度可视化。还集成了新浪财经实时新闻抓取，让系统的输入不局限于本地数据集。

存在的不足：数据集正负样本比例为2.6:1，存在一定的不平衡，可能是模型在少数类上recall略低于多数类的原因。未来可以尝试对负面样本做数据增强，或使用加权损失函数。另外，当前只做了二分类（积极/消极），细粒度情感分析（如强烈看好/中性偏多/中性/中性偏空/强烈看空）会是更有价值的方向。
